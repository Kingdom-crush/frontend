Feasibility

系统适配与技术边界

前端职责

展示项目、规则、执行状态、结果、审计、报告、RTL 图。所有本地敏感能力通过后端 API 完成。

本地后端职责

调用 `spec-hdl` / `vhdl-checker.jar`，解析输出，读写 SQLite，管理文件和插件。

规则集兼容

读取现有 XML；内置规则集只读，自定义规则集允许复制、增删规则、调整等级。

需要确认

麒麟 ARM64 引擎、报告插件接口、软件锁 SDK、RTL 图生成来源。

建议落地顺序

  1. MVP：项目、规则、执行、结果、审计。
  2. 兼容层：导入旧 `toif.db`，输出新 `results.db`。
  3. 插件层：报告生成、软件锁授权、RTL 图生成。
  4. 跨平台打包：Windows x86、Linux x86、麒麟 ARM64 分别验证。
